Tips for Finding Documents and Related Information

The list of documents on this page are PDF files found locally in your documentation directory (usually at C:\CCStudio\docs\pdf ). If for some reason, a link calls up an error message, it may be because the PDF file is not there or mislabeled. You have two options:

  1. copy the PDF file from the install CD (i.e., C:\CCSTudio\docs\pdf).
  2. go to the http://www.dspvillage.com/ and enter your search term in the Keyword field on the top right corner of the page. Entering the document number (i.e., SPRA601 ) tends to bring the most accurate results, although if that fails, you may need to enter the Title or keywords. Many PDF document have alphabet letters to indicate versions. So SPRU018D is more recent than SPRU018C.

If you're not sure what document you need and just want to browse through a list, you can find on the ti.com site a list of online manuals.

Adobe Acrobat Reader 6.0 includes several features to improve the overall reading experience for longer documents(including better search and indexing capability). If you haven't upgraded already, you should do so.

Many of the PDF's linked on this page are lengthy, so it's quite possible that the document encompasses many more topics than the title would suggest. If you're looking for a specific keyword (like a function or plugin tool), you can also click the Index or Search tab on the left side of this help window. Often this can be the fastest way to find the information you need.

Finally, TI now has a searchable DSP knowledgebase which often contains the most up-to-date information about potential bugs and workarounds. Because the number of results may sometimes be unwieldy, it's often helpful to filter results. For example, changing the Document Type to "Frequently Asked Questions" will exclude PDF's and show only questions and answers entered by support engineers. If you have a general question about Code Composer Studio, you can narrow your search by choosing the right DSP Platform (C5000, C6000, etc) but leaving the default "Not answered" for Tool Type. At the time of this writing, search results are NOT organized chronologically, so you may need to be careful that what you are reading is in fact the most recent document.
